# Security Policy - AIC-Recovery

AIC-Recovery is a critical resilience layer designed to protect the entire AIC/HMN ecosystem. Security is not only technical — it is ethical and existential.

### Responsible Disclosure
If you discover a security vulnerability, please report it responsibly.

**How to report:**
- Email: `security@adaptiveintelligencecircle.org`
- Do **NOT** open public GitHub issues.

We will acknowledge your report within 7 days and work to resolve critical issues as quickly as possible.

### Our Security Commitments
- No backdoors or centralized control points.
- All recovery mechanisms are governed by the ethical kernel (IBCS).
- Quantum-resistant practices where applicable.
- Fail-safe design: ethical integrity always takes priority over speed of recovery.
- During the Founder’s military service (2027–2029), security updates will be handled by trusted maintainers under strict review.

### Scope
This policy applies to the entire AIC-Recovery repository and its integration with other AIC components.

**Note:** AIC-Recovery follows zero-donation and third path principles. No financial rewards are offered for security reports.
